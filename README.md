# Крестики-Нолики с ИИ

Обычная игра в крестики-нолики, где можно играть с другом или потягаться с компьютером. ИИ использует алгоритм Minimax, поэтому он практически не сдаёт позиции.

---

## Как запустить

Откройте терминал в папке проекта и напишите:

```bash
python3 -m http.server 8000
```

Потом откройте браузер и перейдите на `http://localhost:8000`

---

## Как это работает

### Стартовый экран

Вводишь имена двух игроков (или своё имя, если играешь с ИИ), выбираешь режим и начинаешь игру.

### Игровой процесс

Доска 3×3. Ты ставишь крестики, противник ставит нолики. Нужно собрать три в ряд — по горизонтали, вертикали или диагонали.

Когда играешь с ИИ, он думает 0,5 секунды перед каждым ходом.

## О коде

### Основные функции

**`resetGame()`** — очищает доску, сбрасывает ход на начало, закрывает окно результата.

**`updateCurrentTurn()`** — обновляет текст "Ход игрока: X/O".

**`handleGameEnd()`** — проверяет, есть ли победитель или ничья, потом переключает игрока или заканчивает игру.

**`checkWinner(board)`** — универсальная функция проверки победителя. Работает как с доской-массивом (для ИИ), так и с DOM-доской.

### Как ИИ делает ходы

ИИ использует алгоритм **Minimax**. Вот пошагово:

#### Шаг 1: `aiMove()`

ИИ смотрит на текущую доску и начинает анализировать возможные ходы.

```javascript
function aiMove() {
  const boardArray = Array.from(cells).map((cell) => cell.textContent);
  const bestMoveIndex = getBestMove(boardArray);
  cells[bestMoveIndex].textContent = currentPlayer;
  handleGameEnd();
}
```

#### Шаг 2: `getBestMove(board)`

Для каждой свободной клетки на доске ИИ проверяет: "А что будет, если я поставлю сюда?" Каждый вариант оценивается с помощью Minimax. Выбирается ход с наивысшей оценкой.

```javascript
function getBestMove(board) {
  let bestScore = -Infinity;
  let bestMoveIndex = -1;

  for (let i = 0; i < 9; i++) {
    if (board[i] === "") {
      board[i] = "O";
      let score = minimax(board, 0, false);
      board[i] = "";

      if (score > bestScore) {
        bestScore = score;
        bestMoveIndex = i;
      }
    }
  }
  return bestMoveIndex;
}
```

#### Шаг 3: `minimax(board, depth, isMaximizing)`

Это рекурсивная функция, которая симулирует игру до конца и оценивает исходы.

**Как она работает:**

1. Если ИИ выиграет — возвращает +10 (минус количество ходов, чтобы предпочесть быструю победу)
2. Если противник выиграет — возвращает -10 (плюс количество ходов)
3. Если ничья — возвращает 0

```javascript
function minimax(board, depth, isMaximizing) {
  const winner = checkWinner(board);

  if (winner === "O") return 10 - depth;
  if (winner === "X") return depth - 10;
  if (winner === "tie") return 0;

  if (isMaximizing) {
    // ИИ выбирает ход с максимальной оценкой
    let bestScore = -Infinity;
    for (let i = 0; i < 9; i++) {
      if (board[i] === "") {
        board[i] = "O";
        let score = minimax(board, depth + 1, false);
        board[i] = "";
        bestScore = Math.max(score, bestScore);
      }
    }
    return bestScore;
  } else {
    // Противник выбирает ход с минимальной оценкой
    let bestScore = Infinity;
    for (let i = 0; i < 9; i++) {
      if (board[i] === "") {
        board[i] = "X";
        let score = minimax(board, depth + 1, true);
        board[i] = "";
        bestScore = Math.min(score, bestScore);
      }
    }
    return bestScore;
  }
}
```

### Почему Minimax работает

ИИ рассчитывает все возможные ходы обоих игроков до конца партии и выбирает самый выгодный для себя. При этом он предполагает, что противник тоже играет оптимально.

В итоге ИИ либо выигрывает, либо играет в ничью. Он никогда не проигрывает.

## Оптимизация кода

Код написан без дублирования:

- Функция **`resetGame()`** используется в двух кнопках вместо копирования одного и того же блока
- Функция **`checkWinner()`** работает и с массивом, и с DOM (вместо двух похожих функций)
- Логика конца хода в **`handleGameEnd()`** (вместо повтора в разных местах)
- Массив выигрышных комбинаций **`WIN_PATTERNS`** — один на всех

## Технические детали

**Сложность алгоритма:** O(9!) в худшем случае, но на практике дерево ходов обрезается рано.

**Время анализа:** обычно 50–200мс, поэтому игра отзывчива.

**Браузеры:** любые современные (Chrome, Firefox, Safari, Edge).
